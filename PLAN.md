# 嘲讽日报 / Satire Daily

每日聚合"外媒嘲讽中国"与"中国官媒嘲讽海外"的现成报道，左右对擂呈现。
本站仅做内容搬运与聚合，不创作讽刺内容，版权及立场归属原媒体。

---

## 一、项目定位

- **形式**：静态站点 + 每日定时抓取脚本
- **受众**：公开但低调运营，境外部署，不留主体信息
- **内容性质**：舆情聚合站，搬运已发布媒体报道，非自创讽刺
- **核心卖点**：借嘴说话——左右栏天然镜像"对方说我坏话"，戏剧性来自对比而非创作

## 二、内容规则

### 2.1 左栏：外媒嘲讽国内
- 抓取源（定向筛选"涉华报道/评论"）：
  - BBC 中文、RFI 法广、DW 德国之声
  - NYT / WSJ 中国版块
  - The Economist 涉华报道
  - Reuters / AP 批评性涉华稿
  - Reddit r/China、r/sino（二次搬运）
- 原文为英文的，**统一用中文呈现**（摘要用中文翻译）

### 2.2 右栏：中国官媒嘲讽海外
- 抓取源（定向筛选"涉外报道/评论"）：
  - 环球时报（中文 + Global Times 英文版）
  - 观察者网涉外评论
  - 参考消息"外媒如何看待"栏目
  - 人民日报海外版、CGTN、中国日报
  - 央视/新华网"国际锐评"类

### 2.3 来源标注（强制）
每条卡片必须包含：
- 来源媒体名
- 原文发布日期
- 原文链接（可点击跳转）
- 来源国旗/标识
- 主题标签（政治/社会/科技/经济/…）

页脚固定声明："本站为内容聚合，仅做摘要与链接搬运，版权及立场归属原媒体，不代表本站观点。"

### 2.4 配对策略
- **弱配对**：左右栏独立，不要求同一事件
- 每边"有几条发几条"，不强凑数量
- 若出现同一事件两边互相嘲讽，标记为"今日对擂"置顶（稀有但价值高）

### 2.5 抓取深度
- **深入正文摘要**：不只抓标题/RSS，要进正文
- 用 LLM 从正文生成 1-2 句忠实中文摘要（不歪曲原意）
- 摘要保留核心事实与嘲讽点，但不得改写为标题党

## 三、技术架构

### 3.1 工作流
```
每日定时 cron
  → 抓取脚本（定向源：外媒涉华 + 中媒涉外）
    ├─ 列表页抓取 → 正文页深入抓取
    └─ 反爬应对（UA、限速、必要时用 headless 浏览器）
  → LLM 处理
    ├─ 分类：是否在嘲讽对方？是→入库；否→丢弃
    ├─ 英文正文→中文摘要翻译
    └─ 打标签：主题（政治/社会/科技/经济）
  → 去重（同一事件多源报道合并/择优）
  → 生成每日数据文件（含原文链接、来源、日期、主题标签）
  → Astro 重新构建静态页
  → 部署到静态托管
```

### 3.2 技术栈
- **站点**：Astro（content collections 渲染每日数据）
- **数据格式**：每日一文件 `src/content/daily/2026-07-19.json`
  ```json
  {
    "date": "2026-07-19",
    "left": [
      {
        "source": "BBC 中文",
        "source_country": "uk",
        "source_url": "https://...",
        "published": "2026-07-19",
        "topic": "社会",
        "title_cn": "...",
        "summary_cn": "...（1-2句中文摘要）"
      }
    ],
    "right": [ ... ],
    "head_to_head": null
  }
  ```
- **爬虫**：Python 脚本（requests/httpx + selectolax 或 playwright 备用）
- **LLM**：Claude / OpenAI / DeepSeek（境外 key 优先）
- **存储**：每日 JSON 文件入 Git（既是数据也是归档），无需数据库
- **部署**：Cloudflare Pages / Vercel + GitHub Actions 自动构建

### 3.3 安全阀（即便全自动也要有）
- LLM 分类阶段直接过滤"非嘲讽对方"的内容
- 摘要阶段用第二遍 LLM 校验：摘要是否歪曲原意
- 关键词扫描黑名单（极端情况兜底）
- 被拦内容存到 `drafts/` 待人工抽查

## 四、部署与合规

- **域名**：境外注册商（Cloudflare/Porkbun），`.com`，不备案
- **托管**：Cloudflare Pages（免费、不留主体信息）
- **爬虫运行位置**：境外 VPS（Contabo/Hetzner 等）
- **Git 仓库**：私有
- **免责声明**：页脚固定显示，明示"聚合搬运，立场归属原媒体"
- **版权**：仅摘要 + 原文链接，不全文搬运
